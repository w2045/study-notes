## 1. 程序是什么？—— 一个精确的视角

在开始写代码之前，我们先建立一个正确的思维框架。Python 程序本质上由两种东西交替构成：

1. **表达式 (expression)**：可以被求值 (evaluate) 的代码片段，求值后产生一个值
2. **语句 (statement)**：执行一个动作，自身不产生值

这个区分非常关键。混淆表达式和语句是人类学习编程时最常见的错误源。

```python
>>> 2 + 3              # 表达式：求值得到 5
5
>>> x = 2 + 3          # 语句：把右边表达式的值绑定给 x
>>>                    # ↑ 注意：赋值语句本身不显示任何东西
```

当你打开 Python 交互环境 (`python3`)，看到 `>>>` 提示符时，Python 处于**读-求值-打印循环** (REPL：Read-Eval-Print Loop)。你输入一行，Python 读它 → 求值它 → 打印结果。如果是语句，打印步骤没有输出。

## 2. 基本数据类型——值的种类

Python 中所有的值都有**类型**。类型决定了你可以对这个值做什么操作。用 `type()` 查看任意值的类型。

### 2.1 整数 `int`

```python
>>> type(42)
<class 'int'>
>>> 2 ** 100                    # Python 整数无限精度
1267650600228229401496703205376
```

**常见运算**：

| 运算 | 符号 | 例子 | 结果 |
|------|------|------|------|
| 加法 | `+` | `2 + 3` | `5` |
| 减法 | `-` | `7 - 4` | `3` |
| 乘法 | `*` | `6 * 7` | `42` |
| 真除法 | `/` | `10 / 4` | `2.5` |
| 整数除法 | `//` | `10 // 4` | `2` |
| 取余 | `%` | `10 % 4` | `2` |
| 乘方 | `**` | `2 ** 10` | `1024` |

### 2.2 浮点数 `float`

```python
>>> type(3.14)
<class 'float'>
>>> 0.1 + 0.2                    # 浮点误差！
0.30000000000000004
```

**重要**：`0.1 + 0.2` 不等于 `0.3`，这是因为 `0.1` 和 `0.2` 在二进制中无法精确表示。**永远不要用 `==` 直接比较两个浮点数**。改用 `abs(a - b) < 1e-9` 或 `math.isclose(a, b)`。

```python
>>> import math
>>> math.isclose(0.1 + 0.2, 0.3)
True
```

### 2.3 字符串 `str`

```python
>>> type("你好")
<class 'str'>
>>> '单引号也可以'
'单引号也可以'
>>> "He said \"Hello!\""        # 用 \" 在里面放引号
'He said "Hello!"'
>>> len("hello")                 # 字符串长度
5
>>> "hello" + " " + "world"      # 拼接
'hello world'
>>> "ha" * 5                     # 重复
'hahahahaha'
```

### 2.4 布尔值 `bool`

布尔类型只有两个值：`True` 和 `False`。它们来自比较运算：

```python
>>> 3 > 2
True
>>> 1 + 1 == 2
True
>>> 5 <= 3
False
>>> True and False               # 逻辑与：两边都 True 才 True
False
>>> True or False                # 逻辑或：只要一边 True 就是 True
True
>>> not True                     # 逻辑非：取反
False
```

比较运算符：`==` (等于), `!=` (不等于), `>`, `<`, `>=`, `<=`。

> **常见错误**：`=` 是赋值，`==` 是比较。写成 `if x = 5:` 会报语法错误。

到目前为止，我们只是在 REPL 里输入表达式，Python 求值后打印结果——然后结果就消失了。想用 `2+3` 的结果做后续计算？没办法，它已经没了。我们需要一种方式**把值存下来**——这就是变量。

## 3. 名字与赋值——变量的本质

### 3.1 赋值语句

```python
x = 10          # 将整数 10 绑定到名字 x
y = x + 5       # 先求值右边（10 + 5 → 15），再绑定给 y
```

赋值是**语句**，不是表达式。它不返回任何值。Python 执行赋值时做了两件事：
1. 求值 `=` 右边的表达式
2. 在当前环境（帧）中创建/更新名字到值的映射

### 3.2 重新赋值

```python
>>> a = 5        # a 指向 5
>>> b = a        # b 也指向 5（不是指向 a！）
>>> a = 7        # a 现在指向 7
>>> b            # b 仍然指向 5
5
```

**关键理解**：`b = a` 不是「b 指向 a」，而是「b 指向 a **当前的值**」。之后修改 `a` 不会影响 `b`。整数是不可变对象——你无法改变 `5` 本身，你只能让名字指向一个新的值。

### 3.3 命名规则

| 合法 | 不合法 | 原因 |
|------|--------|------|
| `my_var` | `2bad` | 数字不能开头 |
| `_hidden` | `class` | 关键字 |
| `camelCase` | `my-var` | 不能有连字符 |
| `名字` | `a b` | 不能有空格 |

Python 关键字一览（不能用作变量名）：`False, None, True, and, as, assert, break, class, continue, def, del, elif, else, except, finally, for, from, global, if, import, in, is, lambda, nonlocal, not, or, pass, raise, return, try, while, with, yield`

### 3.4 求值的完整过程

当 Python 看到 `x = f(y)` 时：
1. 求值 `f` —— 在环境中查找名字 `f`，得到函数对象
2. 求值 `y` —— 在环境中查找名字 `y`，得到它的值
3. **调用** `f`，传入 `y` 的值，得到返回值
4. **绑定** 返回值给名字 `x`

> 这解释了为什么 `a = a + 1` 可行：Python 先用 `a` 的旧值算出 `a+1`，再把结果重新绑定到 `a`。

变量可以存储值，但程序不只是存储值——还需要**重复执行操作**。比如你想算一个数的平方，写 `x * x` 没问题；但要在十个地方算十个不同数的平方，每次都手写 `x * x` 就麻烦了。我们需要一种方式**把一段计算打包，起个名字，随时调用**——这就是函数。

## 4. 函数的定义与调用

### 4.1 `def` 语句解剖

```python
def square(x):
    """返回 x 的平方。"""   # ← 文档字符串 (docstring)
    return x * x            # ← 函数体，缩进 4 个空格
```

| 部分 | 名称 | 说明 |
|------|------|------|
| `def` | 关键字 | 告诉 Python 这是一个函数定义 |
| `square` | 函数名 | 你给函数取的名字 |
| `(x)` | 参数列表 | `x` 是**形式参数** (parameter)，占位符 |
| `: ` | | 函数头结束 |
| 缩进块 | 函数体 | 调用时执行的代码 |
| `return` | 返回语句 | 把计算结果交给调用者 |

**类型注解**（Python 3.5+，推荐）：

```python
def square(x: float) -> float:
    """返回 x 的平方。"""
    return x * x
```

类型注解只是文档，Python 不会强制检查。但它们让你的函数意图更清晰。

### 4.2 参数 vs 实参

- **形式参数 (parameter)**：定义时的占位名 → `def f(x, y)` 中的 `x, y`
- **实际参数 (argument)**：调用时传入的值 → `f(3, 5)` 中的 `3, 5`

调用时，实参按位置依次绑定到形参。这称为**按位置传参**。

```python
def power(base, exp):
    return base ** exp

power(2, 10)   # base=2, exp=10 → 1024
power(10, 2)   # base=10, exp=2 → 100  # 不同！
```

### 4.3 默认参数

```python
def greet(name, greeting="Hello"):
    """greeting 有默认值 'Hello'。"""
    return f"{greeting}, {name}!"

greet("张三")                   # "Hello, 张三!"
greet("李四", greeting="你好")   # "你好, 李四!"
```

有默认值的参数放在**无默认值的参数之后**，否则语法错误。

### 4.4 局部变量与作用域

函数内部赋值的变量是**局部变量**——它只存在于函数调用期间。

```python
def demo():
    local_var = 42   # 局部变量
    return local_var

demo()
# print(local_var)  # 报错！NameError: name 'local_var' is not defined
```

这引出一个重要原理：**函数调用创建独立的名字空间**。不同函数调用之间、函数与外部之间是隔离的。

```
全局帧：
  demo → [函数对象]
  x → 5

调用 demo() 时创建局部帧：
  局部帧（demo 的）：
    local_var → 42
  ↑ 函数返回后销毁
```

### 4.5 `return` vs `print` —— 初学者的分水岭

这是 Python 学习中最容易被误解的一组概念。

```python
def add_and_print(a, b):
    print("calculating...")     # 在屏幕上显示 → 这是副作用
    return a + b                # 把值交给调用者 → 这是真正的输出

r = add_and_print(3, 4)          # 屏幕显示 "calculating..."
print(r)                         # 屏幕显示 7
```

| | `return` | `print` |
|---|---|---|
| **目的** | 把值传给程序的其他部分 | 把文字显示给人看 |
| **值去哪了** | 成为调用表达式的结果 | 出现在终端/控制台 |
| **之后能用吗** | 可以赋值给变量 | 不能，它只是显示 |
| **没有时** | 函数返回 `None` | 什么都没显示 |

```
           ┌──────────┐
           │  函数 f   │
           │          │
 print ────┤ → 屏幕   │   (副作用，给人看)
           │          │
 return ───┤ → 调用者 │   (真正的输出，给程序用)
           └──────────┘
```

## 5. 环境模型（Environment Model）

上一节我们学会了定义和调用函数。但几个关键问题还没回答：函数内部的变量（局部变量）存在哪里？为什么函数外面访问不到它们？为什么 `a = a + 1` 可以工作——Python 怎么区分「右边的旧 a」和「左边的新 a」？

环境模型是 CS61A 中最核心的思维工具，它用一个统一的机制回答以上所有问题。掌握它之后，后续的闭包、递归、可变性——这些「诡异」的现象——都会变得可以精确解释。

### 5.1 帧 (Frame) 与环境

- **帧**：一张表，存储「名字 → 值」的映射。你可以把它想成字典 `{name: value}`
- **环境**：一列帧的链，从内向外。最外层是全局帧，它包含所有不在函数内定义的名字
- **当前帧** 总有对**父帧**的引用，形成一条链

```
全局帧
┌──────────────────────────────┐
│ print → <built-in function>  │
│ abs   → <built-in function>  │
│ x     → 10                   │
│ f     → func f(x) [parent=全局] │
└──────────────────────────────┘
```

### 5.2 查找规则

当 Python 遇到一个名字时：
1. 先在**当前帧**中查找
2. 找不到就去**父帧**（外层），依次向外
3. 直到全局帧，还没找到就报 `NameError`

### 5.3 函数调用的完整步骤

调用 `f(3)` 时：
1. 求值操作符 `f` → 在环境中找到函数对象
2. 求值操作数 `3` → 得到值 3
3. **创建局部帧**，其父帧是函数定义时所在的环境
4. **绑定形参**：在局部帧中，把形参名绑定到实参值
5. **执行函数体**：在局部帧中逐条执行
6. 遇到 `return` → 返回值给调用处，销毁局部帧
7. 没 `return` → 返回 `None`

**具体走一个例子**：

```python
def f(x):
    return x + y       # y 不在局部帧中，去全局帧找

y = 10
result = f(3)           # result = 13
```

执行 `f(3)` 时：
```
帧链：
  局部帧 (f)             全局帧
  ┌──────────┐          ┌──────────────────┐
  │ x → 3    │──parent→ │ f → func f        │
  │          │          │ y → 10            │
  │ x+y=?    │          │ result → ?        │
  └──────────┘          └──────────────────┘

查找 y：局部帧中没有 → 去全局帧 → 找到 y=10 → 返回 3+10=13
```

## 6. 条件表达式（三元运算符）

环境模型给了我们一个精确的心智模型。在进入第二章完整的控制流（`if`/`while`/`for`）之前，先学一个立即可用的小工具。有时你只需要在一行里根据条件挑一个值——比如根据分数选「及格」还是「不及格」。这时候写完整的 `if-else` 语句太啰嗦，条件表达式一行搞定：

```python
# 语法: value_if_true if condition else value_if_false
>>> "偶数" if 6 % 2 == 0 else "奇数"
'偶数'
>>> "通过" if 85 >= 60 else "不通过"
'通过'
```

这是**表达式**（产生值），不同于 `if` **语句**（只执行动作）。所以你可以在表达式期望的地方用它：

```python
x = "正" if n > 0 else "非正"    # 可以赋值
print(f"{n} 是{'偶数' if n%2==0 else '奇数'}")   # 可以嵌入 f-string
```

## 7. 测试与断言

现在你已经能写表达式、定义变量、创建函数。但写完一个函数后，怎么知道它是对的？靠「看着像」吗？这个函数 `square(5)` 应该返回 25——但万一你手误写成了 `x + x`，它会返回 10 而你完全没发现。你需要一种**自动验证**的机制——每当代码改动，一键确认之前正确的行为没有被破坏。

### 7.1 `assert`

```python
assert 条件, "失败时的消息"
```

- 条件为 `True`：静默通过
- 条件为 `False`：抛出 `AssertionError`，显示消息

```python
def square(x):
    return x * x

assert square(5) == 25
assert square(0) == 0
assert square(-4) == 16, f"square(-4) 应该是 16, 但得到 {square(-4)}"
```

### 7.2 `doctest` —— 把测试写在文档里

```python
def square(x):
    """返回 x 的平方。

    >>> square(5)
    25
    >>> square(0)
    0
    >>> square(-4)
    16
    """
    return x * x

if __name__ == "__main__":
    import doctest
    doctest.testmod()
```

运行 `python3 文件名.py` 会执行文档中的所有测试。如果通过，什么都不显示。如果失败，显示差异。

## 8. 调试策略

`assert` 和 `doctest` 告诉你「某个测试失败了」，但没有告诉你「为什么失败」和「在哪一行失败」。当你盯着一个不工作的函数，不知道从哪下手时，下面三个方法让你快速定位问题。

初学阶段最实用的三个调试手段：

1. **打印中间值**：在怀疑的地方插入 `print(repr(变量))`
2. **缩小范围**：注释掉一半代码，确定 bug 在哪一半，反复二分
3. **检查类型**：`print(type(x), x)` 同时看到类型和值

```python
def mystery(a, b):
    print(f"DEBUG: a={a!r}, b={b!r}")   # !r 显示 repr（更精确）
    result = a / b
    print(f"DEBUG: result={result!r}")
    return result
```

## 本章核心概念速查

| 概念 | 一句话 |
|------|--------|
| 表达式 | 可求值的代码，如 `2+3` → `5` |
| 语句 | 执行动作，不产生值，如 `x = 5` |
| 类型 | `int`, `float`, `str`, `bool` |
| 浮点误差 | `0.1+0.2 ≠ 0.3`，用 `math.isclose` |
| 赋值 | `=` 将名字绑定到值，不是等式 |
| 形式参数 | 定义时的占位名 |
| 实际参数 | 调用时的具体值 |
| 局部变量 | 函数内部的变量，调用结束后消失 |
| 环境 | 帧的链，从内向外查找名字 |
| `return` vs `print` | 前者把值给程序，后者把文字给人 |
| 条件表达式 | `a if cond else b`，是表达式 |
| `doctest` | 把测试嵌入文档字符串 |
